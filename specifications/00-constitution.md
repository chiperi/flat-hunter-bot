# Project Constitution — flat-hunter-bot

Незмінні принципи проєкту (найвищий рівень; на них спираються spec / plan / decisions).
Порушувати їх можна лише через явний ADR, що фіксує причину й наслідки.

1. **Redis-only, без реляційної БД.** Дані плоскі (дедуп + фільтр-профілі), тож PostgreSQL —
   надлишок незалежно від кількості користувачів. Стан «уже бачив» **завжди** в Redis, ніколи
   в пам'яті процесу: рестарт не повинен спричиняти дублі сповіщень. Seen-стан — це **Hash**
   (`listingId → lastPrice`), а не Set, щоб ловити ще й зміну ціни.

2. **Нуль вхідних портів, повна ізоляція.** Бот лише робить вихідні виклики (Telegram long polling +
   сайти оголошень). Жодного webhook, домену чи SSL. На дроплеті — окремий docker-compose стек,
   окрема мережа, без спільних контейнерів із сусіднім проєктом (`aurora`).

3. **Стійкість вища за повноту.** Помилка одного джерела, профілю, користувача чи сповіщення **не
   може** зупинити цикл для інших. Кожне джерело повертає `[]` при збої (ніколи не кидає в цикл);
   кожен профіль і кожне сповіщення — у власному try/catch; глобальний `bot.catch` для інтерактивних
   помилок. «Seen» проставляється **лише після** успішної відправки — збій під час notify лишає
   оголошення неврахованим для ретраю, а не тихо ковтає його.

4. **Один фільтр на користувача — на всі сайти.** `/newsearch` створює/оновлює **один** профіль,
   що опитується проти **кожного** ввімкненого джерела; кожне сповіщення позначене джерелом.
   Дедуп неймспейсний per-site (`source:id`), тож одна квартира на двох сайтах не колізить.

5. **Ціни завжди в гривні, без вигадування.** Валютні ($/€) ціни конвертуються в грн за курсом,
   який дає **саме джерело** (DOM.RIA `priceArr`, Rieltor — НБУ-еквівалент у картці). Якщо
   гривневого значення немає — оголошення без ціни («договірна»), **ніколи** не підписуємо іноземну
   суму як грн. Оголошення без ціни у вибірку не потрапляють.

6. **Скрапінг — делікатний і за абстракцією.** Джерела — за інтерфейсом `ListingSource`/`SiteSpec`,
   щоб підхід (прямий фетч ↔ проксі) мінявся без переписування решти. Джитер інтервалу, retry з
   експоненційним backoff, ротація User-Agent, невелика затримка перед запитом, і **дедуп однакових
   пошуків** в один фетч за цикл. Обсяг запитів свідомо тримаємо малим (напр. DOM.RIA — cap на
   деталі за цикл).

7. **Доступ — за allowlist, fail-closed.** Бот лише для дозволеного кола (`ALLOWED_USER_IDS`),
   перевірка до будь-якої команди. Невідомий користувач отримує ввічливу відмову, не тишу й не
   сирий error. Кожна мутація профілю перевіряє власника (`profile.userId === ctx.from.id`).

8. **Українськомовний інтерфейс за замовчуванням.** Уся копірайт-частина бота — українською.

9. **Свідоме звуження, не тихі поломки.** Наразі **лише Київ** (geo DOM.RIA + крок майстра). Обмеження
   мають бути явними в UI/доках, а не мовчазним «нічого не знайдено».

10. **Зміни — під тестами й CI.** Кожна зміна проходить сюїту та coverage-gate (80/80/80/70) на
    кожному push/PR; деплой на дроплет — лише із зеленого `main` через GitHub Actions по SSH,
    секрети ніколи не пишуться у файл на хості.
