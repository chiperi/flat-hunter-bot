# ADR-0003: Sources landscape — DOM.RIA + Rieltor only

**Status:** accepted
**Date:** 2026-07-09

## Context
Проєкт починався як «OLX-бот». На практиці кожне джерело перевірялося наживо з дроплета (дата-центровий
IP). Результати:
- **OLX / lun.ua / flatfy.ua** — за Cloudflare, віддають `403` з дата-центрового IP (bot-fight);
  lun/flatfy до того ж — агрегатори екосистеми LUN, що дублюють DOM.RIA/rieltor.
- **birdrent.com / josti.com.ua** — це застосунки-сервіси без веб-каталогу оголошень (лише лендінги
  App Store / Google Play; усі URL каталогу → soft-404). Скрапити нема що.
- **DOM.RIA** — має офіційне developer-API (доступ за ключем). **Rieltor** — server-rendered HTML,
  доступний із дроплета без блоку.

## Decision
Живі джерела — **DOM.RIA** (офіційне API) + **Rieltor** (HTML). `KNOWN_SOURCE_IDS = domria, rieltor`.
OLX/lun/flatfy та їхній `__NEXT_DATA__`-парсер-кластер **видалено** з коду (можна відновити з git,
якщо з'явиться резидентський проксі). birdrent/josti відкинуто повністю (нема веб-даних).

## Consequences
- ➕ Код чесно відображає реальність; нема мертвих адаптерів і оманливих «OLX»-згадок.
- ➕ Найлегітимніший доступ: DOM.RIA — за ключем, Rieltor — без обходу блокувань; персональні дані
  (телефони/імена) не зберігаються.
- ➖ Менше покриття, ніж «усі сайти». Розширення (OLX/lun/flatfy) впирається в **один** блокер —
  резидентський/ротаційний проксі (`HTTP_PROXY_URL`); тоді джерела вмикаються без нового коду.
- 🔒 GDPR/ToS: скрапінг Rieltor — сіра зона (без обходу блоку); публічний запуск/монетизація —
  привід для юридичної консультації.
